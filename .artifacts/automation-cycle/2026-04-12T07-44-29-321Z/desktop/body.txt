Auto-dispatch effect

Relevant area:

apps/web/src/components/ChatView.tsx:3753

A useEffect watches queue + runtime conditions and automatically sends the first queued draft when all blockers are clear.

It will not send if any of these are true:

current turn still running
send already busy
connecting
dispatch already in flight
pending approval exists
pending user input exists

When eligible:

it takes queuedFollowUps[0]
sends it
removes it from queue on success
currently also shifts it out on failure after restoring it to composer state

So queued follow-ups are strict FIFO by default, with a manual “send next” reorder control.

Steer now = inject follow-up into the active running turn via streaming API
Queue follow-up = hold a draft locally and send a new turn automatically after the current turn settles
