# Security Policy

If you find a security issue, please do not open a public GitHub issue.

Instead, report it privately to the maintainers with:

- a short description of the issue
- affected versions or commits
- reproduction steps or a proof of concept
- impact assessment
- any suggested mitigation

If this repository is published under an org with GitHub Security Advisories enabled, please prefer private reporting there.

Until a dedicated security contact is documented, assume public issues are not an appropriate place for vulnerability reports.
